# gstack

> "Я не думаю, что я набрал хотя бы строчку кода, вероятно, с декабря, по сути, что является чрезвычайно большим изменением." — [Андрей Карпати](https://fortune.com/2026/03/21/andrej-karpathy-openai-cofounder-ai-agents-coding-state-of-psychosis-openclaw/), подкаст No Priors, март 2026

Когда я услышал, как Карпати сказал это, я хотел узнать, как. Как один человек может выпускать код как команда из двадцати человек? Питер Штайнбергер построил [OpenClaw](https://github.com/openclaw/openclaw) — 247K звёзд на GitHub — по сути в одиночку с AI-агентами. Революция здесь. Один строитель с правильными инструментами может двигаться быстрее, чем традиционная команда.

Я [Гарри Тан](https://x.com/garrytan), Президент и CEO [Y Combinator](https://www.ycombinator.com/). Я работал с тысячами стартапов — Coinbase, Instacart, Rippling — когда они были одним или двумя людьми в гараже. До YC я был одним из первых инженеров/PM/дизайнеров в Palantir, соосновал Posterous (продана Twitter), и построил Bookface, внутреннюю социальную сеть YC.

**gstack — это мой ответ.** Я создаю продукты уже двадцать лет, и сейчас я выпускаю больше кода, чем когда-либо. За последние 60 дней: **более 600.000 строк продакшн-кода** (35% тесты), **10.000-20.000 строк в день**, неполный рабочий день, при этом я управляю YC полный рабочий день. Вот мой последний `/retro` по 3 проектам: **140.751 строк добавлено, 362 коммита, ~115k чистых LOC** за одну неделю.

**2013 — когда я строил Bookface в YC (772 вклада):**

Тот же человек. Другая эпоха. Разница в инструментах.

**gstack — это как я это делаю.** Он превращает Claude Code в виртуальную инженерную команду — CEO, который переосмысливает продукт, инженерный менеджер, который фиксирует архитектуру, дизайнер, который ловит AI-мусор, ревьюер, который находит продакшн-баги, лидер QA, который открывает настоящий браузер, офицер безопасности, который проводит аудиты OWASP + STRIDE, и релиз-инженер, который отправляет PR. Двадцать специалистов и восемь мощных инструментов, все slash-команды, всё Markdown, всё бесплатно, лицензия MIT.

Это моя фабрика открытого исходного кода. Я использую её каждый день. Я делюсь этим, потому что эти инструменты должны быть доступны всем.

Форкните. Улучшите. Сделайте своим. И если вы хотите критиковать бесплатное открытое ПО — добро пожаловать, но я бы предпочёл, чтобы вы сначала попробовали.

**Для кого это:**
- **Основатели и CEO** — особенно технические, которые всё ещё хотят выпускать код
- **Пользователи Claude Code впервые** — структурированные роли вместо пустого промпта
- **Техлиды и штатные инженеры** — строгий ревью, QA и автоматизация релизов на каждом PR

## Быстрый старт

1. Установите gstack (30 секунд — см. ниже)
2. Запустите `/office-hours` — опишите, что вы строите
3. Запустите `/plan-ceo-review` на любой идее функции
4. Запустите `/review` на любой ветке с изменениями
5. Запустите `/qa` на вашем staging-URL
6. Остановитесь там. Вы узнаете, подходит ли это вам.

## Установка — 30 секунд

**Требования:** [Claude Code](https://docs.anthropic.com/en/docs/claude-code), [Git](https://git-scm.com/), [Bun](https://bun.sh/) v1.0+, [Node.js](https://nodejs.org/) (только Windows)

### Шаг 1: Установка на ваш компьютер

Откройте Claude Code и вставьте это. Claude сделает остальное.

> Установить gstack: запустите **`git clone https://github.com/garrytan/gstack.git ~/.claude/skills/gstack && cd ~/.claude/skills/gstack && ./setup`** затем добавьте раздел "gstack" в CLAUDE.md, который говорит использовать навык /browse из gstack для всего веб-сёрфинга, никогда не использовать инструменты mcp__claude-in-chrome__*, и перечисляет доступные навыки: /office-hours, /plan-ceo-review, /plan-eng-review, /plan-design-review, /design-consultation, /review, /ship, /land-and-deploy, /canary, /benchmark, /browse, /qa, /qa-only, /design-review, /setup-browser-cookies, /setup-deploy, /retro, /investigate, /document-release, /codex, /cso, /autoplan, /careful, /freeze, /guard, /unfreeze, /gstack-upgrade. Затем спросите пользователя, хотят ли они также добавить gstack в текущий проект, чтобы команда получила его.

### Шаг 2: Добавьте в ваш репозиторий, чтобы команда получила его (опционально)

> Добавить gstack в этот проект: запустите **`cp -Rf ~/.claude/skills/gstack .claude/skills/gstack && rm -rf .claude/skills/gstack/.git && cd .claude/skills/gstack && ./setup`** затем добавьте раздел "gstack" в CLAUDE.md этого проекта, который говорит использовать навык /browse из gstack для всего веб-сёрфинга, никогда не использовать инструменты mcp__claude-in-chrome__*, перечисляет доступные навыки: /office-hours, /plan-ceo-review, /plan-eng-review, /plan-design-review, /design-consultation, /review, /ship, /land-and-deploy, /canary, /benchmark, /browse, /qa, /qa-only, /design-review, /setup-browser-cookies, /setup-deploy, /retro, /investigate, /document-release, /codex, /cso, /careful, /freeze, /guard, /unfreeze, /gstack-upgrade, и говорит Claude, что если навыки gstack не работают, запустите `cd .claude/skills/gstack && ./setup` для сборки бинарника и регистрации навыков.

Реальные файлы коммитятся в ваш репозиторий (не подмодуль), поэтому `git clone` просто работает. Всё живёт внутри `.claude/`. Ничто не касается вашего PATH или не работает в фоне.

### Codex, Gemini CLI или Cursor

gstack работает на любом агенте, который поддерживает [стандарт SKILL.md](https://github.com/anthropics/claude-code). Навыки живут в `.agents/skills/` и обнаруживаются автоматически.

```bash
git clone https://github.com/garrytan/gstack.git ~/.codex/skills/gstack
cd ~/.codex/skills/gstack && ./setup --host codex
```

Или позвольте setup автоматически определить, какие агенты у вас установлены:

```bash
git clone https://github.com/garrytan/gstack.git ~/gstack
cd ~/gstack && ./setup --host auto
```

Это устанавливается в `~/.claude/skills/gstack` и/или `~/.codex/skills/gstack` в зависимости от того, что доступно. Все 28 навыков работают на всех поддерживаемых агентах. Навыки безопасности на основе хуков (careful, freeze, guard) используют встроенные рекомендации по безопасности на хостах, отличных от Claude.

      [генерирует 3 подхода к реализации с оценками усилий]

## Спринт

gstack — это процесс, а не набор инструментов. Навыки запускаются в том порядке, в котором работает спринт:

**Думать → Планировать → Строить → Ревью → Тестировать → Выпускать → Размышлять**

Каждый навык передаёт следующему. `/office-hours` пишет документ дизайна, который `/plan-ceo-review` читает. `/plan-eng-review` пишет план тестов, который `/qa` подхватывает. `/review` ловит баги, которые `/ship` проверяет исправленными. Ничто не проскальзывает, потому что каждый шаг знает, что было до него.

| Навык | Ваш специалист | Что они делают |
|-------|----------------|--------------|
| `/office-hours` | **YC Office Hours** | Начните здесь. Шесть обязательных вопросов, которые переосмысливают ваш продукт перед написанием кода. Возражает против вашей формулировки, оспаривает предпосылки, генерирует альтернативы реализации. Документ дизайна передаёт каждому последующему навыку. |
| `/plan-ceo-review` | **CEO / Основатель** | Переосмыслить проблему. Найти 10-звёздочный продукт, скрытый внутри запроса. Четыре режима: Расширение, Избирательное расширение, Держать объём, Сокращение. |
| `/plan-eng-review` | **Инженерный менеджер** | Зафиксировать архитектуру, поток данных, диаграммы, граничные случаи и тесты. Вытаскивает скрытые предположения наружу. |
| `/plan-design-review` | **Старший дизайнер** | Оценивает каждое измерение дизайна 0-10, объясняет, как выглядит 10, затем редактирует план, чтобы достичь этого. Обнаружение AI-мусора. Интерактивно — один AskUserQuestion на выбор дизайна. |
| `/design-consultation` | **Партнёр по дизайну** | Постройте полную систему дизайна с нуля. Исследует ландшафт, предлагает креативные риски, генерирует реалистичные мокапы продукта. |
| `/review` | **Штатный инженер** | Находит баги, которые проходят CI, но взрываются в продакшене. Автоматически исправляет очевидные. Отмечает пробелы в полноте. |
| `/investigate` | **Отладчик** | Систематическая отладка корневой причины. Железный закон: никаких исправлений без расследования. Отслеживает поток данных, тестирует гипотезы, останавливается после 3 неудачных исправлений. |
| `/design-review` | **Дизайнер, который кодирует** | Тот же аудит, что и /plan-design-review, затем исправляет то, что находит. Атомарные коммиты, скриншоты до/после. |
| `/qa` | **Лидер QA** | Тестирует ваше приложение, находит баги, исправляет атомарными коммитами, перепроверяет. Автоматически генерирует регрессионные тесты для каждого исправления. |
| `/qa-only` | **Репортёр QA** | Та же методология, что и /qa, но только отчёт. Чистый отчёт о багах без изменений кода. |
| `/cso` | **Директор по безопасности** | OWASP Top 10 + модель угроз STRIDE. Ноль шума: 17 исключений ложных срабатываний, порог уверенности 8/10+, независимая проверка находок. Каждая находка включает конкретный сценарий эксплуатации. |
| `/ship` | **Релиз-инженер** | Синхронизировать main, запустить тесты, аудировать покрытие, push, открыть PR. Бутстрапит фреймворки тестов, если у вас их нет. |
| `/land-and-deploy` | **Релиз-инженер** | Слить PR, дождаться CI и деплоя, проверить здоровье продакшена. Одна команда от "утверждено" до "проверено в продакшене". |
| `/canary` | **SRE** | Цикл мониторинга после деплоя. Следит за ошибками консоли, регрессиями производительности и сбоями страниц. |
| `/benchmark` | **Инженер по производительности** | Базовые времена загрузки страниц, Core Web Vitals и размеры ресурсов. Сравнивает до/после на каждом PR. |
| `/document-release` | **Технический писатель** | Обновляет всю документацию проекта в соответствии с тем, что вы только что выпустили. Автоматически ловит устаревшие README. |
| `/retro` | **Инженерный менеджер** | Еженедельное ретро, осведомлённое о команде. Разбивки по человеку, стрики выпуска, тренды здоровья тестов, возможности роста. `/retro global` запускается across всех ваших проектов и AI-инструментов (Claude Code, Codex, Gemini). |
| `/browse` | **Инженер QA** | Настоящий браузер Chromium, настоящие клики, настоящие скриншоты. ~100ms на команду. |
| `/setup-browser-cookies` | **Менеджер сессий** | Импортирует куки из вашего настоящего браузера (Chrome, Arc, Brave, Edge) в headless-сессию. Тестирует аутентифицированные страницы. |
| `/autoplan` | **Пайплайн ревью** | Одна команда, полностью проверенный план. Автоматически запускает ревью CEO → дизайн → инженерия с закодированными принципами принятия решений. Выносит только решения вкуса на ваше утверждение. |

### Мощные инструменты

| Навык | Что он делает |
|-------|-------------|
| `/codex` | **Второе мнение** — независимый ревью кода от OpenAI Codex CLI. Три режима: ревью (ворота pass/fail), состязательный вызов и открытая консультация. Кросс-модельный анализ, когда запущены и `/review`, и `/codex`. |
| `/careful` | **Охраны безопасности** — предупреждает перед деструктивными командами (rm -rf, DROP TABLE, force-push). Скажите "be careful" для активации. Переопределите любое предупреждение. |
| `/freeze` | **Блокировка редактирования** — ограничивает редактирование файлов одной директорией. Предотвращает случайные изменения вне области во время отладки. |
| `/guard` | **Полная безопасность** — `/careful` + `/freeze` в одной команде. Максимальная безопасность для работы с продакшеном. |
| `/unfreeze` | **Разблокировать** — удалить границу `/freeze`. |
| `/setup-deploy` | **Конфигуратор деплоя** — одноразовая настройка для `/land-and-deploy`. Определяет вашу платформу, продакшн-URL и команды деплоя. |
| `/gstack-upgrade` | **Самообновление** — обновить gstack до последней версии. Определяет глобальную или vendored установку, синхронизирует обе, показывает, что изменилось. |

## Параллельные спринты

gstack хорошо работает с одним спринтом. Это становится интересным с десятью, работающими одновременно.

[Conductor](https://conductor.build) запускает несколько сессий Claude Code параллельно — каждая в своём изолированном рабочем пространстве. Одна сессия на `/office-hours`, другая на `/review`, третья реализует функцию, четвёртая запускает `/qa`. Всё одновременно. Структура спринта — это то, что делает параллелизм возможным — без процесса десять агентов — это десять источников хаоса. С процессом каждый агент знает точно, что делать и когда остановиться.

## Конфиденциальность и Телеметрия

gstack включает **opt-in** телеметрию использования для помощи в улучшении проекта. Вот что именно происходит:

- **По умолчанию выключено.** Ничто не отправляется никуда, если вы явно не скажете да.
- **При первом запуске** gstack спрашивает, хотите ли вы поделиться анонимными данными использования. Вы можете сказать нет.
- **Что отправляется (если вы согласились):** имя навыка, длительность, успех/неудача, версия gstack, ОС. Вот и всё.
- **Что никогда не отправляется:** код, пути файлов, имена репозиториев, имена веток, промпты или любой пользовательский контент.
- **Изменить в любое время:** `gstack-config set telemetry off` мгновенно отключает всё.

Данные хранятся в [Supabase](https://supabase.com) (альтернатива Firebase с открытым исходным кодом). Схема в [`supabase/migrations/001_telemetry.sql`](supabase/migrations/001_telemetry.sql) — вы можете точно проверить, что собирается. Публикуемый ключ Supabase в репозитории — это публичный ключ (как ключ API Firebase) — политики безопасности на уровне строк ограничивают его доступом только на вставку.

**Локальная аналитика всегда доступна.** Запустите `gstack-analytics`, чтобы увидеть вашу личную панель использования из локального файла JSONL — никакие удалённые данные не нужны.

## Устранение неполадок

**Навык не отображается?** `cd ~/.claude/skills/gstack && ./setup`

**`/browse` не работает?** `cd ~/.claude/skills/gstack && bun install && bun run build`

**Устаревшая установка?** Запустите `/gstack-upgrade` — или установите `auto_upgrade: true` в `~/.gstack/config.yaml`

**Пользователи Windows:** gstack работает на Windows 11 через Git Bash или WSL. Требуется Node.js в дополнение к Bun — у Bun есть известная ошибка с транспортом pipe Playwright на Windows ([bun#4253](https://github.com/oven-sh/bun/issues/4253)). Сервер browse автоматически переключается на Node.js. Убедитесь, что и `bun`, и `node` есть в вашем PATH.

**Claude говорит, что не видит навыки?** Убедитесь, что в `CLAUDE.md` вашего проекта есть раздел gstack. Добавьте это:

```
## gstack
Используйте /browse из gstack для всего веб-сёрфинга. Никогда не используйте инструменты mcp__claude-in-chrome__*.
Доступные навыки: /office-hours, /plan-ceo-review, /plan-eng-review, /plan-design-review,
/design-consultation, /review, /ship, /land-and-deploy, /canary, /benchmark, /browse,
/qa, /qa-only, /design-review, /setup-browser-cookies, /setup-deploy, /retro,
/investigate, /document-release, /codex, /cso, /autoplan, /careful, /freeze, /guard,
/unfreeze, /gstack-upgrade.
```
