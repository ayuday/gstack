# gstack

> "ฉันไม่คิดว่าฉันได้พิมพ์โค้ดแม้แต่บรรทัดเดียวตั้งแต่เดือนธันวาคม โดยพื้นฐานแล้ว ซึ่งเป็นการเปลี่ยนแปลงที่ใหญ่มาก" — [Andrej Karpathy](https://fortune.com/2026/03/21/andrej-karpathy-openai-cofounder-ai-agents-coding-state-of-psychosis-openclaw/), พอดแคสต์ No Priors, มีนาคม 2026

เมื่อฉันได้ยิน Karpathy พูดแบบนี้ ฉันต้องการหาว่าเป็นอย่างไร คนหนึ่งคนสามารถส่งมอบงานได้เหมือนทีมยี่สิบคนได้อย่างไร? Peter Steinberger สร้าง [OpenClaw](https://github.com/openclaw/openclaw) — 247K ดาวบน GitHub — โดยพื้นฐานแล้วคนเดียวด้วยเอเจนต์ AI การปฏิวัติอยู่ที่นี่ ผู้สร้างคนเดียวที่มีเครื่องมือที่เหมาะสมสามารถเคลื่อนไหวได้เร็วกว่าทีมแบบดั้งเดิม

ฉันคือ [Garry Tan](https://x.com/garrytan), ประธานและ CEO ของ [Y Combinator](https://www.ycombinator.com/) ฉันเคยทำงานกับสตาร์ทอัพหลายพันแห่ง — Coinbase, Instacart, Rippling — เมื่อพวกเขายังเป็นหนึ่งหรือสองคนในโรงรถ ก่อน YC ฉันเป็นหนึ่งในวิศวกร/PM/ดีไซเนอร์คนแรกที่ Palantir ร่วมก่อตั้ง Posterous (ขายให้ Twitter) และสร้าง Bookface เครือข่ายสังคมภายในของ YC

**gstack คือคำตอบของฉัน** ฉันสร้างผลิตภัณฑ์มายี่สิบปี และตอนนี้ฉันกำลังส่งมอบโค้ดมากกว่าที่เคย ใน 60 วันที่ผ่านมา: **โค้ดการผลิตมากกว่า 600,000 บรรทัด** (35% เป็นการทดสอบ), **10,000-20,000 บรรทัดต่อวัน**, พาร์ทไทม์, ในขณะที่บริหาร YC ฟูลไทม์ นี่คือ `/retro` ล่าสุดของฉันใน 3 โปรเจกต์: **เพิ่ม 140,751 บรรทัด, 362 commits, สุทธิ ~115k LOC** ในหนึ่งสัปดาห์

**gstack คือวิธีที่ฉันทำ** มันเปลี่ยน Claude Code ให้เป็นทีมวิศวกรรมเสมือน — CEO ที่คิดทบทวนผลิตภัณฑ์ใหม่ ผู้จัดการวิศวกรรมที่ล็อกสถาปัตยกรรม ดีไซเนอร์ที่จับผิดงาน AI ขยะ ผู้ตรวจสอบที่พบบั๊กในโปรดักชัน ผู้นำ QA ที่เปิดเบราว์เซอร์จริง เจ้าหน้าที่ความปลอดภัยที่รันการตรวจสอบ OWASP + STRIDE และวิศวกรปล่อยที่ส่ง PR ผู้เชี่ยวชาญยี่สิบคนและเครื่องมือทรงพลังแปดอย่าง ทั้งหมดเป็นคำสั่งสแลช ทั้งหมดเป็น Markdown ทั้งหมดฟรี ใบอนุญาต MIT

นี่คือโรงงานซอฟต์แวร์โอเพนซอร์สของฉัน ฉันใช้มันทุกวัน ฉันแบ่งปันมันเพราะเครื่องมือเหล่านี้ควรพร้อมสำหรับทุกคน

Fork มัน ปรับปรุงมัน ทำให้มันเป็นของคุณ และถ้าคุณต้องการเกลียดชังซอฟต์แวร์โอเพนซอร์สฟรี — ยินดีต้อนรับ แต่ฉันอยากให้คุณลองก่อน

**สำหรับใคร:**
- **ผู้ก่อตั้งและ CEO** — โดยเฉพาะด้านเทคนิคที่ยังต้องการส่งมอบ
- **ผู้ใช้ Claude Code ครั้งแรก** — บทบาทที่มีโครงสร้างแทนที่จะเป็น prompt ว่างเปล่า
- **Tech leads และ staff engineers** — การตรวจสอบที่เข้มงวด QA และระบบอัตโนมัติการปล่อยในทุก PR

## เริ่มต้นอย่างรวดเร็ว

1. ติดตั้ง gstack (30 วินาที — ดูด้านล่าง)
2. รัน `/office-hours` — อธิบายสิ่งที่คุณกำลังสร้าง
3. รัน `/plan-ceo-review` บนไอเดียฟีเจอร์ใดๆ
4. รัน `/review` บน branch ใดๆ ที่มีการเปลี่ยนแปลง
5. รัน `/qa` บน URL staging ของคุณ
6. หยุดตรงนั้น คุณจะรู้ว่านี่เหมาะกับคุณหรือไม่

## การติดตั้ง — 30 วินาที

**ข้อกำหนด:** [Claude Code](https://docs.anthropic.com/en/docs/claude-code), [Git](https://git-scm.com/), [Bun](https://bun.sh/) v1.0+, [Node.js](https://nodejs.org/) (เฉพาะ Windows)

### ขั้นตอนที่ 1: ติดตั้งบนเครื่องของคุณ

เปิด Claude Code และวางสิ่งนี้ Claude ทำส่วนที่เหลือ

> ติดตั้ง gstack: รัน **`git clone https://github.com/garrytan/gstack.git ~/.claude/skills/gstack && cd ~/.claude/skills/gstack && ./setup`** จากนั้นเพิ่มส่วน "gstack" ใน CLAUDE.md ที่บอกว่าใช้ทักษะ /browse จาก gstack สำหรับการท่องเว็บทั้งหมด อย่าใช้เครื่องมือ mcp__claude-in-chrome__* และแสดงรายการทักษะที่มี: /office-hours, /plan-ceo-review, /plan-eng-review, /plan-design-review, /design-consultation, /review, /ship, /land-and-deploy, /canary, /benchmark, /browse, /qa, /qa-only, /design-review, /setup-browser-cookies, /setup-deploy, /retro, /investigate, /document-release, /codex, /cso, /autoplan, /careful, /freeze, /guard, /unfreeze, /gstack-upgrade จากนั้นถามผู้ใช้ว่าต้องการเพิ่ม gstack ในโปรเจกต์ปัจจุบันเพื่อให้เพื่อนร่วมทีมได้รับด้วยหรือไม่

### ขั้นตอนที่ 2: เพิ่มลงใน repo ของคุณเพื่อให้เพื่อนร่วมทีมได้รับ (ไม่บังคับ)

> เพิ่ม gstack ในโปรเจกต์นี้: รัน **`cp -Rf ~/.claude/skills/gstack .claude/skills/gstack && rm -rf .claude/skills/gstack/.git && cd .claude/skills/gstack && ./setup`** จากนั้นเพิ่มส่วน "gstack" ใน CLAUDE.md ของโปรเจกต์นี้ที่บอกว่าใช้ทักษะ /browse จาก gstack สำหรับการท่องเว็บทั้งหมด อย่าใช้เครื่องมือ mcp__claude-in-chrome__* แสดงรายการทักษะที่มี: /office-hours, /plan-ceo-review, /plan-eng-review, /plan-design-review, /design-consultation, /review, /ship, /land-and-deploy, /canary, /benchmark, /browse, /qa, /qa-only, /design-review, /setup-browser-cookies, /setup-deploy, /retro, /investigate, /document-release, /codex, /cso, /careful, /freeze, /guard, /unfreeze, /gstack-upgrade และบอก Claude ว่าถ้าทักษะ gstack ไม่ทำงาน ให้รัน `cd .claude/skills/gstack && ./setup` เพื่อสร้าง binary และลงทะเบียนทักษะ

ไฟล์จริงจะถูก commit ลงใน repo ของคุณ (ไม่ใช่ submodule) ดังนั้น `git clone` จึงใช้งานได้ทันที ทุกอย่างอยู่ใน `.claude/` ไม่มีอะไรสัมผัส PATH ของคุณหรือรันในพื้นหลัง

### Codex, Gemini CLI หรือ Cursor

gstack ทำงานบนเอเจนต์ใดๆ ที่รองรับ [มาตรฐาน SKILL.md](https://github.com/anthropics/claude-code) ทักษะอยู่ใน `.agents/skills/` และถูกค้นพบโดยอัตโนมัติ

```bash
git clone https://github.com/garrytan/gstack.git ~/.codex/skills/gstack
cd ~/.codex/skills/gstack && ./setup --host codex
```

หรือให้ setup ตรวจจับโดยอัตโนมัติว่าคุณติดตั้งเอเจนต์ใดบ้าง:

```bash
git clone https://github.com/garrytan/gstack.git ~/gstack
cd ~/gstack && ./setup --host auto
```

สิ่งนี้ติดตั้งใน `~/.claude/skills/gstack` และ/หรือ `~/.codex/skills/gstack` ขึ้นอยู่กับสิ่งที่พร้อมใช้งาน ทักษะทั้ง 28 ทำงานบนเอเจนต์ที่รองรับทั้งหมด ทักษะความปลอดภัยที่ใช้ hook (careful, freeze, guard) ใช้คำแนะนำความปลอดภัยแบบ inline บนโฮสต์ที่ไม่ใช่ Claude

## Sprint

gstack คือกระบวนการ ไม่ใช่ชุดเครื่องมือ ทักษะทำงานตามลำดับที่ sprint ทำงาน:

**คิด → วางแผน → สร้าง → ตรวจสอบ → ทดสอบ → ส่งมอบ → ไตร่ตรอง**

แต่ละทักษะป้อนทักษะถัดไป `/office-hours` เขียนเอกสารการออกแบบที่ `/plan-ceo-review` อ่าน `/plan-eng-review` เขียนแผนการทดสอบที่ `/qa` รับไป `/review` จับบั๊กที่ `/ship` ตรวจสอบว่าแก้ไขแล้ว ไม่มีอะไรหลุดรอดเพราะทุกขั้นตอนรู้ว่าอะไรอยู่ก่อนหน้า

| ทักษะ | ผู้เชี่ยวชาญของคุณ | สิ่งที่พวกเขาทำ |
|-------|----------------|--------------|
| `/office-hours` | **YC Office Hours** | เริ่มที่นี่ หกคำถามบังคับที่กรอบผลิตภัณฑ์ของคุณใหม่ก่อนเขียนโค้ด คัดค้านการกรอบของคุณ ท้าทายสมมติฐาน สร้างทางเลือกการดำเนินการ เอกสารการออกแบบป้อนทุกทักษะถัดไป |

## Sprint ขนาน

gstack ทำงานได้ดีกับ sprint หนึ่ง มันน่าสนใจเมื่อสิบรันพร้อมกัน

[Conductor](https://conductor.build) รันเซสชัน Claude Code หลายเซสชันแบบขนาน — แต่ละเซสชันในพื้นที่ทำงานที่แยก出自己的หนึ่งเซสชันที่ `/office-hours` อีกเซสชันที่ `/review` เซสชันที่สามดำเนินการฟีเจอร์ เซสชันที่สี่รัน `/qa` ทั้งหมดในเวลาเดียวกัน โครงสร้าง sprint คือสิ่งที่ทำให้การทำงานขนานเป็นไปได้ — โดยไม่มีกระบวนการ เอเจนต์สิบตัวคือสิบแหล่งของความโกลาหล ด้วยกระบวนการ แต่ละเอเจนต์รู้ точноว่าต้องทำอะไรและเมื่อไหร่ต้องหยุด

## ความเป็นส่วนตัวและการวัดทางไกล

gstack รวมการวัดการใช้งาน **opt-in** เพื่อช่วยปรับปรุงโปรเจกต์ นี่คือสิ่งที่เกิดขึ้น точно:

- **ปิดโดยค่าเริ่มต้น** ไม่มีการส่งอะไรไปไหนเว้นแต่คุณจะพูดว่าใช่อย่างชัดเจน
- **ในการรันแรก** gstack ถามว่าคุณต้องการแชร์ข้อมูลการใช้งานที่ไม่ระบุชื่อหรือไม่ คุณสามารถพูดว่าไม่
- **สิ่งที่ส่ง (ถ้าคุณเลือกเข้าร่วม):** ชื่อทักษะ ระยะเวลา ความสำเร็จ/ล้มเหลว เวอร์ชัน gstack OS เท่านั้น
- **สิ่งที่ไม่มีวันส่ง:** โค้ด เส้นทางไฟล์ ชื่อ repo ชื่อ branch prompts หรือเนื้อหาที่ผู้ใช้สร้างใดๆ
- **เปลี่ยนได้ตลอดเวลา:** `gstack-config set telemetry off` ปิดทุกอย่างทันที

ข้อมูลถูกเก็บใน [Supabase](https://supabase.com) (ทางเลือกโอเพนซอร์สของ Firebase) สคีมาอยู่ใน [`supabase/migrations/001_telemetry.sql`](supabase/migrations/001_telemetry.sql) — คุณสามารถตรวจสอบ точноว่าเก็บอะไร คีย์ที่เผยแพร่ได้ของ Supabase ใน repo เป็นคีย์สาธารณะ (เหมือนคีย์ API ของ Firebase) — นโยบายความปลอดภัยระดับแถวจำกัดเป็นการเข้าถึงแบบใส่เท่านั้น

**การวิเคราะห์ในเครื่องพร้อมใช้งานเสมอ** รัน `gstack-analytics` เพื่อดูแดชบอร์ดการใช้งานส่วนตัวของคุณจากไฟล์ JSONL ในเครื่อง — ไม่ต้องใช้ข้อมูลระยะไกล

## การแก้ไขปัญหา

**ทักษะไม่แสดง?** `cd ~/.claude/skills/gstack && ./setup`

**`/browse` ล้มเหลว?** `cd ~/.claude/skills/gstack && bun install && bun run build`

**การติดตั้งล้าสมัย?** รัน `/gstack-upgrade` — หรือตั้งค่า `auto_upgrade: true` ใน `~/.gstack/config.yaml`

**ผู้ใช้ Windows:** gstack ทำงานบน Windows 11 ผ่าน Git Bash หรือ WSL ต้องการ Node.js นอกจาก Bun — Bun มีบั๊กที่รู้จักกับการขนส่ง pipe ของ Playwright บน Windows ([bun#4253](https://github.com/oven-sh/bun/issues/4253)) เซิร์ฟเวอร์ browse ถอยกลับเป็น Node.js โดยอัตโนมัติ ตรวจสอบว่าทั้ง `bun` และ `node` อยู่ใน PATH ของคุณ

**Claude บอกว่ามองไม่เห็นทักษะ?** ตรวจสอบว่า `CLAUDE.md` ของโปรเจกต์คุณมีส่วน gstack เพิ่มสิ่งนี้:

```
## gstack
ใช้ /browse จาก gstack สำหรับการท่องเว็บทั้งหมด อย่าใช้เครื่องมือ mcp__claude-in-chrome__*
ทักษะที่มี: /office-hours, /plan-ceo-review, /plan-eng-review, /plan-design-review,
/design-consultation, /review, /ship, /land-and-deploy, /canary, /benchmark, /browse,
/qa, /qa-only, /design-review, /setup-browser-cookies, /setup-deploy, /retro,
/investigate, /document-release, /codex, /cso, /autoplan, /careful, /freeze, /guard,
/unfreeze, /gstack-upgrade
```
